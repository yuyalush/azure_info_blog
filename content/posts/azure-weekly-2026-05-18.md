今週のハイライト
- Azure Kubernetes Fleet ManagerでのCiliumベースのクロスクラスターネットワーキングの導入
- EDAワークロード向けのAzure NetApp Filesの新機能
- Azure IaaSにおけるシステムレベルアプローチによる高性能ワークロードの展開
- Azure FilesにおけるEntra-Onlyアイデンティティの一般提供開始

## コンピューティング
### Powering multi-cluster workloads with seamless cross‑cluster networking for Azure Kubernetes Fleet Manager
Azure Kubernetes Fleet ManagerにCiliumベースのクロスクラスターネットワーキングが導入され、全体的なフリートを横断する高性能なマネージドネットワークが実現されました。これにより、マルチクラスターワークロードの効率的な運用が可能になります。詳細は[こちら](https://azure.microsoft.com/en-us/blog/powering-multi-cluster-workloads-with-seamless-cross-cluster-networking-for-azure-kubernetes-fleet-manager/)。

### Azure IaaS: Deploy high-performance workloads with a system-level approach
Azure IaaSは、計算、ストレージ、ネットワーキングの統合によるシステムレベルアプローチを採用し、一貫したスケーラブルなパフォーマンスを実現します。これにより、AIやクラウドネイティブ、ビジネスクリティカルなワークロードにおいて高性能なワークロードの展開が可能となります。詳細は[こちら](https://azure.microsoft.com/en-us/blog/azure-iaas-deploy-high-performance-workloads-with-a-system-level-approach/)。

## ストレージ
### Azure NetApp Files for EDA workloads: From revolution to breakthrough at scale
Azure NetApp Filesは、EDA（電子設計自動化）ワークロード向けに、スケーラブルで高性能なストレージソリューションを提供します。これにより、大規模な同時実行、低遅延、一貫した生産性を実現し、従来のストレージボトルネックを克服できます。詳細は[こちら](https://azure.microsoft.com/en-us/blog/azure-netapp-files-for-eda-workloads-from-revolution-to-breakthrough-at-scale/)。

## セキュリティ
### Azure Files Entra-Only identities: Advancing cloud-native identity and security
Azure Filesでは、Entra-Onlyアイデンティティの一般提供が開始され、Microsoft Entra IDによるネイティブ認証をサポートします。これにより、クラウドネイティブなアイデンティティを利用したSMBファイル共有への安全なアクセスが可能になります。詳細は[こちら](https://azure.microsoft.com/en-us/blog/azure-files-entra-only-identities-advancing-cloud-native-identity-and-security/)。
